# Code of Conduct

This project follows the Contributor Covenant Code of Conduct, version 2.1.

## Our Standard

Participants are expected to keep discussion technical, respectful, and focused on improving the project.

Examples of constructive behavior include:

- explaining tradeoffs with evidence;
- reviewing code and documentation with specific references;
- respecting operational safety constraints around credentials, billing, and destructive actions;
- assuming public issues may be read by future users and contributors.

## Enforcement

Maintainers may remove comments, close threads, or restrict participation when behavior is abusive, harassing, or repeatedly disruptive.

For sensitive reports, use the repository security advisory workflow rather than posting exploit or credential details publicly.
